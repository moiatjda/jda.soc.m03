################
Petita correcció
################

Ara ja sabem fer el programa corresponent depenent del nombre de valors
que hàgim de demanar.

Si ens fixem, però, això no és ben bé el que ens demanava l'enunciat del
problema original. Tal i com l'estem resolent, el programa demana sempre **tots**
els valors. És a dir, no s'atura quan llegeix el primer negatiu!

El que voldríem és quelcom similar al següent diagrama:

.. graphviz::
    :align: center

        inici[shape="box", style=rounded];
        final[shape="box", style=rounded];
        instruccio0 [label="suma = 0", shape="rectangle", style=""];
        instruccio3 [label="escriu suma", shape="rectangle", style=""]


        compound=true;

            lectura1 [label="valor = llegeix(\"un valor\")", shape="rectangle", style=""];

És a dir, en cas que el primer número llegit sigui negatiu, ja no
demanarem el segon!

Això encara ho podem fer amb els coneixements que tenim! Com?

Fàcil, posem el bloc de processament del segon valor dins del primer
condicional!

El codi resultant seria:

.. code-block:: python
    :linenos:

    """
        Programa que suma nombres positius consecutius, fins 
        rebre un de negatiu
        Versió simplificada: fins a dos valors
    """

    # creem la variable on guardarem el resultat de sumar
    suma = 0   # inicialment no hem sumat res i per tant és 0

    # processem el primer valor
    print("Introdueix un valor")
    valor = int(input())
    if valor >= 0:
        suma = suma + valor

        # processem el segon valor
        print("Introdueix un valor")
        valor = int(input())
        if valor >= 0:
            suma = suma + valor

    # mostrem el resultat
    print("La suma és", suma)

Fixa't com, ara només llegim el segon valor si hem sumat el primer, és a
dir, si el primer no era negatiu!
